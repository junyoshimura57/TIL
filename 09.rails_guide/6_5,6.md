gem## 現場rails（P.53~）
- nilガード  
`number || = 10`は`number || (number= 10)`と同じ動きをして「numberが真であればnumeberを返し、偽(nilかfalse)の場合はnumberに10を入れて返す」という意味になる。  

- ぼっち演算子  
`&.`を使ってメソッドを呼び出すとレシーバーがnilの場合でもエラーが発生しない。  
以下は全て同じ意味となる。  
```
name = if object
  object.name
else
  nil
end
```
```
name = ? ojbect.name : nil
```
```
name = ojbect&.name
```
